# Story 5.4: Reserve Report Drafting Through the Gate

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As an Analyst,
I want a drafted Reserve Report whose every claim is cited and every figure engine-rendered,
so that my starting draft is already audit-grade. (FR-11)

## Acceptance Criteria

1. **Given** accepted recommendations, **When** the agent drafts the report, **Then** the draft contains the four sections — executive summary, method selection rationale, movement commentary, limitations — with figures only as `{{rs:...}}` placeholders and claims citing `{{dx:...}}` chips, rendered and verified by the Provenance Gate before persistence (FR-11, AD-5).
2. **And** a draft failing the gate is redrafted within the bounded attempt limit; exhaustion fails the Interpretation cleanly (no partial report is ever persisted).
3. **Given** a gated-passing draft, **When** persisted, **Then** it is stored as a Reserve Report draft linked to the Run with machine-drafted provenance marked, and the drafting transcript audit-logged (FR-11, FR-15).
4. **And** Interpretation completes within the ≤10-minute bound (NFR-7) or fails cleanly — never a silent queue (the synchronous succeed-or-fail path; the hard timeout budget is Story 5.6).

    1. Build a fresh agent per attempt: `build_interpretation_agent(model, result_set, diagnostics_bundle, instructions=build_report_prompt(result_set, diagnostics_bundle, recommendations))`; on a redraft, append the prior attempt's rejection reasons to the user prompt (the model is told exactly what failed — empty section X, unsourced number in section Y, uncited claim in section Z — so it can fix it).     2. `result = run_interpretation(agent, prompt)`; capture `result.transcript` for this attempt regardless of outcome.
    3. `parse_report_draft(result.output_text)` → on `DraftParseError`, record the attempt with a `draft_unparseable` rejection (`source="parse"`) and continue.
    4. **Gate each section** through `run_provenance_gate(section_text, result_set, diagnostics_bundle)` (5.2), iterating the four `(name, raw_text)` pairs. (This is AC-1: no section becomes accepted content without passing the gate; a figure not written as `{{rs:...}}` fails `unsourced_number`, an uncited quantitative block fails `uncited_claim`.)     5. When all four sections gate-passed, assemble the candidate `ReserveReport(run_id=diagnostics_bundle.run_id, machine_drafted=True, executive_summary=…, method_selection_rationale=…, movement_commentary=…, limitations=…)`, then `validate_reserve_report(candidate, result_set, diagnostics_bundle)` (Task 1.3) for the structural guarantees (non-empty sections, resolvable citations). Combine gate rejections + structural rejections into the attempt's rejection set.
    6. If the attempt has **zero** rejections → `ReserveReportAccepted(report=candidate, attempts=...)`, return immediately. Else record the `AttemptRecord` and loop.

## Dev Notes

### Architecture compliance (non-negotiable)

- **AD-1 / The Constitution — the LLM never calculates.** The report's every figure is a `{{rs:...}}` placeholder the gate renders from the ResultSet, never a literal the model computed. `validate_reserve_report` and the loop compute NO reserve figures (no sums/deltas/ratios). The report is machine-drafted content governed by the gate (AD-5) — persist only what passed. The prompt must NOT feed the model literal figures (not even from the accepted recommendations' rendered reasons — see §Feeding recommendations without literals).
- **AD-5 — Provenance Gate governs machine-drafted content.** Every section runs through `run_provenance_gate` (5.2) before it can become an accepted `ReserveReportSection`: unsourced numbers → `unsourced_number`, uncited quantitative blocks → `uncited_claim`, unresolvable citations → `unresolvable_dx_citation`, bad placeholders → `malformed_placeholder`/`cross_run_placeholder`/`unresolvable_rs_placeholder`. A draft that fails is never returned/persisted as reviewable; the rejection (with the transcript) is audit-logged and drives the bounded redraft. Prompt instructions (Task 3) are NOT a substitute for the programmatic gate.
- **AD-8 — read-only agent over Agno/google-genai.** The agent reaches evidence ONLY through the four read-only tools (5.1); the report prompt is provider-neutral; the single provider seam is `build_gemini_model`. Model id from `engine_service` config. **No new tool** — the accepted `Recommendations` are fed via the prompt (as method choices, not as a tool), so the read-only tool surface is unchanged (list_diagnostics / get_diagnostic / get_result_fields / get_run_metadata). No write, no network beyond the model plane.
- **AD-3 — Convex is the sole system of record; engine_service is stateless & never calls Convex.** The Run's ResultSet + DiagnosticsBundle + Recommendations are passed DOWN into `/reports` (not fetched by the engine); the accepted ReserveReport + the transcript are returned UP to the Convex action, which persists + audits. No durable Agno session state — redraft context is re-supplied via the prompt.
- **AD-6 / FR-15 — every LLM interaction is audit-logged.** Each attempt's full transcript rides in the audit payload — on success (`report.drafted`) AND on failure (`report.draftRejected`). `appendAuditEntry` is the sole audit writer; the chain stays append-only + hash-chained.
- **AD-4 — no anonymous Convex access.** `generateReserveReport` / `getReserveReport` call `requireMember` first. Approve/publish/override (senior_actuary) is Epic 6.
- **AD-9 — fails closed into Engine-Only Mode.** `ModelNotConfiguredError` → typed `model_unavailable` (not retried, not swallowed); Story 5.6 derives Engine-Only Mode from it. The per-Run cost ceiling + the ≤10-min interpretation timeout are 5.6's; here `max_attempts` is the shared constant and NFR-7 (AC-4) is satisfied structurally by the synchronous succeed-or-fail path (no silent queue).
- **AD-10 — camelCase cross-runtime contract, drift-checked.** The `ReserveReport` model is schema-exported and its Convex validator is drift-checked in CI exactly like ResultSet/DiagnosticsBundle/Recommendations. Diagnostic IDs are resolved via `resolve_diagnostic`, never string-split.

### Persistence shape — dedicated `reserveReports` table (reversing 5.3's inline decision)

Story 5.3 deliberately persisted `recommendations` **inline** on the `runs` row (like `resultSet`/`diagnosticsBundle`) and flagged: "If Epic 6 later needs recommendations versioned independently of the Run … revisit then." **For the Reserve Report, that time is now.** The Reserve Report is not regenerable machine-only state — the PRD (§4.5) calls it "a human-owned artifact from the moment it exists," and Epic 6 requires:
- an **independent status lifecycle** distinct from `run.status`: `draft` → `awaiting review` → `published` (6.2/6.4);
- **immutable published versions** with a "Start new version" superseding record (6.4, FR-13);
- **human-edit content versioning** and edit audit with content versions (6.1).

None of that fits an inline optional on `runs` (which has exactly one value, overwritten in place). A dedicated `reserveReports` table keyed by `runId` (+ a `by_workspace` index for the Epic-6/7 review queue) is the right home, and it is a *deliberate, documented reversal* of 5.3's inline choice — not an inconsistency. 5.4 ships it **lean**: `status` is a single `draft` literal (Epic 6 extends the union), no versioning columns yet (Epic 6 owns them), one row per run that a re-draft overwrites. `report` is typed by `reserveReportValidator` so a schema-invalid document can never be stored (AD-10). This mirrors how `runs` itself was born minimal and grew — the table exists now so Epic 6 extends columns rather than migrating a document out of `runs`.

`getRun` gains a `hasReserveReport` boolean via one `by_run` indexed read (not a denormalised flag on `runs` — that would duplicate state; the extra read keeps `getRun` lean and figure-free, consistent with `hasResults`/`hasDiagnostics`/`hasRecommendations`).

### The redraft loop belongs in engine_service (same as 5.3)

`report_flow.generate_reserve_report` puts the bounded redraft loop **in engine_service**, exactly as `recommendations_flow` does — a redraft is an agent turn that must see what failed and try again within one request, preserving the transient Agno session semantics (AD-3) and keeping the agent + gate co-located in the shell that hosts them (AD-2/AD-8). The Convex `generateReserveReport` action stays the thin persist+audit tail (`generateRecommendations` is the exact template). Both arms audit every attempt's transcript + rejections. This is the established 5.3 pattern, not a new decision — reuse it verbatim.

### Feeding recommendations without literals (AD-1 subtlety)

The report is drafted "Given accepted recommendations" (AC-1), so `build_report_prompt` takes the `Recommendations` document. BUT the recommendations' `reasons[].text` are **post-gate rendered strings that contain literal figures** (the gate already substituted `{{rs:...}}` → display values in 5.3). If those strings were dumped into the report prompt verbatim, the model would naturally copy a literal figure into a report section, and the gate would reject it as `unsourced_number` (a fabricated-but-coincidental literal is the one accepted residual — see 5.2 deferred note — but most copied literals won't be in the source-value set for the section's context, and even those that are defeat the "figures come from placeholders" intent). So the prompt feeds the model only the **method choice per Origin Period** (`origin → method`) and optionally the **set of Diagnostic IDs already cited per origin** (opaque `dx:` ids, no figures), then instructs it to write every figure fresh as an `{{rs:...}}` placeholder. Task 8.3 asserts a recommendation-reason literal does NOT appear in the prompt. This keeps AD-1 airtight even though a rendered document is an input.

### Section citation minima (why `ReserveReportSection.citations` may be empty)

`RecommendationReason` requires `≥1` citation (a recommendation reason with no evidence is meaningless). A **report section** is different: the `limitations` section may legitimately contain a purely-qualitative caveat with no figure and thus no citation, and forcing `≥1` structural citation per section would wrongly reject it. The Provenance Gate's per-block `uncited_claim` rule already enforces the real invariant — **any block that states a figure must carry a citation** — at the granularity that matters. So `validate_reserve_report` does NOT require `≥1` citation per section; it requires non-empty sections + resolvable citations, and lets the gate own the claim-citation coupling. (FR-11's "every claim cites a Diagnostic ID" = every *quantitative* claim, which the gate enforces.) This is a genuine divergence from the recommendations validator; documented so a reviewer doesn't "fix" it into a false rejection.
