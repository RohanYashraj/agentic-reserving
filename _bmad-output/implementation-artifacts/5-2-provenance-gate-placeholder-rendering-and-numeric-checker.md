## Story

As a Senior Actuary,
I want a programmatic gate that renders engine values into drafts and rejects any unsourced number,
so that no machine-drafted figure can ever reach a reviewer unverified. (FR-11 gate, AD-5)

## Acceptance Criteria

1. **Given** a machine draft containing placeholders, **When** the gate renders it, **Then** `{{rs:<runId>:<method>:<origin>:<field>}}` tokens resolve from the ResultSet and `{{dx:<diagnosticId>}}` tokens resolve to citation references; any unresolvable placeholder fails the gate.
2. **And** the numeric-token checker verifies every numeric token in the rendered output matches a source value under a documented canonicalization rule (rounding/formatting), with a whitelist for structural numerals (headings, dates) (AD-5).
3. **Given** a failing draft, **When** the gate rejects it, **Then** the draft is never persisted as reviewable, the rejection with reasons is audit-logged, and the caller receives a typed rejection enabling a bounded redraft loop (FR-11).
4. **And** pytest covers: clean pass, literal-number smuggling (LLM emitting a figure directly), placeholder pointing at the wrong field, mismatched rounding, uncited quantitative claim, and whitelist correctness.

## Dev Notes

### Architecture compliance (non-negotiable)

- **AD-5 â€” Provenance Gate: template injection first, numeric checker second (the whole point of this story).** Render `{{rs:...}}`/`{{dx:...}}` from the ResultSet/DiagnosticsBundle FIRST, then run the numeric-token checker over the rendered output. Every numeric token must match a source value under the **documented canonicalization rule** (Task 1.2 â€” round half-to-even to 2 dp, no separators) or be a whitelisted structural numeral; every quantitative claim must cite â‰¥1 resolvable Diagnostic ID. A failing draft is **never** produced as accepted `rendered_content`; the rejection is a typed value the caller audit-logs. Prompt instructions are not a substitute for this programmatic gate.
- **AD-1 â€” the LLM never calculates; numbers originate only in the engine.** The gate READS engine values and canonicalizes/formats them for display and comparison â€” it computes **no** reserve figure (no sums, deltas, ratios). `engine_source_values` only collects and normalizes numbers that already exist on the frozen models. Display formatting (thousands separators) is the one permitted non-engine transform ("display formatting only"), and it must be provably the same precision as the canonicalization key.
- **AD-2 / layering â€” the gate is a pure module in the imperative shell.** It lives in `engine_service` (spine: "imperative shell + provenance gate") but performs no I/O, HTTP, Convex, Clerk, filesystem, clock, or logging â€” so it is directly unit-testable and safe inside stateless request handling. It imports only `reserving_engine` (downward) + stdlib. It must NOT import `copilot_agent` (the gate guards the agent's output from outside â€” the agent cannot gate itself) or `engine_service.app` (no FastAPI coupling). `uv run lint-imports` stays green (gate sits in the `engine_service` layer, imports the core beneath it).
- **AD-3 â€” Convex is the sole system of record; engine_service never calls Convex.** The gate does not persist or audit-log; it returns a typed `GateRejected`/`GateAccepted`. The caller (a Convex action, via the 5.3/5.4 endpoint) audit-logs the rejection and drives the bounded redraft loop. Nothing durable here.
- **AD-10 â€” the wire is the camelCase contract.** The rejection/accepted models reuse `_MODEL_CONFIG` (camelCase, `frozen`), Diagnostic IDs are resolved by `resolve_diagnostic` (NEVER string-split â€” labels are opaque and contain `:`), and `rs` field names are the camelCase `OriginResult` wire names (`ultimate`, `ibnr`, `mackStdErr`, `reserveLow`, `reserveHigh`). The placeholder `<method>` is the ResultSet method literal (snake_case: `chain_ladder`).

### The canonicalization rule (read this â€” it is the crux of the checker)

The gate's guarantee rests on one comparison key: a draft number is "sourced" iff `canonicalize_number(token) âˆˆ engine_source_values(result_set, bundle)`.

- **Rule (documented, single source of truth in the module docstring):** parse â†’ `Decimal` â†’ quantize round-**half-to-even** to **2 dp** â†’ normalized plain-decimal string, no grouping separators, `-0 â†’ 0`, trailing `.00`/`.x0` normalized (so `18834.0 â†’ "18834"`, `1.0234 â†’ "1.02"`).
- The **renderer** display-formats (`format_figure`: `18834.0 â†’ "18,834"`) and the **checker** strips separators before canonicalizing, so display and comparison reconcile by construction â€” keep the precision constant shared between `format_figure` and `canonicalize_number`.
- **Why 2 dp:** it is the tolerance. Taylor-Ashe figures are integer-valued (round-trip exact), and diagnostics ratios/residuals reduce to 2 dp â€” enough to make "mismatched rounding" (a token at the wrong precision) fall out of the source set while not rejecting a correctly-quoted figure. Flagged tunable in deferred-work if a future currency needs sub-cent precision.
- **Accepted residual (document, don't try to solve in 5.2):** a fabricated number that *coincidentally* canonicalizes to a real engine value passes. AD-5 forbids *unsourced* numbers reaching a reviewer; a number equal to an engine value is sourced-equivalent/verifiable, so this is a deliberate, bounded acceptance â€” not a hole.

### The placeholder grammar (exact shapes the tests build against)

- `{{rs:<runId>:<method>:<origin>:<field>}}` â†’ figure. `runId` MUST equal `bundle.run_id` (else `cross_run_placeholder`). Resolve `MethodResult(method) â†’ OriginResult(origin) â†’ field`; missing/None â†’ `unresolvable_rs_placeholder`. Renders the display-formatted figure.
- `{{dx:<diagnosticId>}}` â†’ citation. Inner id is the whole opaque `dx:{runId}:{kind}:{key}` (contains `:`). `resolve_diagnostic(bundle, id)`; miss â†’ `unresolvable_dx_citation`.
